Why care about computer networks?

When you visit a website what happens? How does a set of texts, images, fonts, videos, etc. suddenly appear on your screen? Is it wizardry and why is it important for web developers to learn? Shouldn't web developers focus their attention on how to create websites not how they are loaded into a browser?

All valid questions that ultimately lead to one answer...

It's important to have a strong foundation of the engine you are working on.

In my limited experience as a web developer there are two types of websites one can make:

  • static websites
  • dynamic websites

As a beginner you will be largely making static websites which involve little to zero knowledge about computer networks. As you progress, however, you may want to update the content of your website based on specific conditions from data/information delivered from a server.

Websites that update their content based on conditions from the data received from a server are called dynamic websites and make up most of the websites you interact with in your day to day life.

As a web developer you are going to be writing JavaScript code that makes requests to servers for data/information like so:

// get html element that contains all the posts
const posts = document.querySelector("#posts");

// make request to server
fetch("myserver.com/posts")
.then (async response => {
    if (response.status == 400) {
        console.error("Bad Request")
    } else {
        await return response.json();
    }
})
.then(data => {
    // go through every post
    for (const post of data.posts) {
        // create html element to display post description on webpage
        const post = document.createElement("p");
        post.innerText = post.desc;
        // add post to html webpage
        posts.appendChild(post)
    }
})

Now depending on your experience with web development, the code above may seem like gibberish. I have bad news... it will get even more complicated than that as you will have to deal with request headers, actions, body data, etc. Then you'll have to prepare for different server response codes!

So, learning about computer networks is imperative to being a successful web developer! Let's start with defining what a computer network is!

A high-level view of computer networks

A computer network is a system that allows computing devices to communicate with each other. To provide a practical example, say you are using a laptop that is connected to a router. That laptop and other devices connected to the router are part of a web known as a computer network.

They can communicate with each other, but how?

As mentioned earlier a computer network allows computing devices to communicate with each other. In order for this to occur the computing devices on the network must speak the same language.

Similarly to how in order for humans to understand each other we must speak or understand the same language.

  • Computing devices on a computer network must use a common communications protocol (wiki.)
    • Communications Protocol

      Set of rules that describe how information is to be exchanged between entities
