Why care about computer networks?

When you visit a website what happens? How does a set of texts, images, fonts, videos, etc. suddenly appear on your screen? Is it wizardry and why is it important for web developers to learn? Shouldn't web developers focus their attention on how to create websites not how they are loaded into a browser?

All valid questions that ultimately lead to one answer...

It's important to have a strong foundation of the engine you are working on.

In my limited experience as a web developer there are two types of websites one can make:

  • static websites
  • dynamic websites

As a beginner you will be largely making static websites which involve little to zero knowledge about computer networks. As you progress, however, you may want to update the content of your website based on specific conditions from data/information delivered from a server.

Websites that update their content based on conditions from the data received from a server are called dynamic websites and make up most of the websites you interact with in your day to day life.

As a web developer you are going to be writing JavaScript code that makes requests to servers for data/information like so:

// get html element that contains all the posts
const posts = document.querySelector("#posts");

// make request to server
fetch("myserver.com/posts")
.then (async response => {
    if (response.status == 400) {
        console.error("Bad Request")
    } else {
        await return response.json();
    }
})
.then(data => {
    // go through every post
    for (const post of data.posts) {
        // create html element to display post description on webpage
        const post = document.createElement("p");
        post.innerText = post.desc;
        // add post to html webpage
        posts.appendChild(post)
    }
})

Now depending on your experience with web development, the code above may seem like gibberish. I have bad news... it will get even more complicated than that as you will have to deal with request headers, actions, body data, etc. Then you'll have to prepare for different server response codes!

So, learning about computer networks is imperative to being a successful web developer! Let's start with defining what a computer network is!

A high-level view of computer networks

A computer network is a system that allows computing devices to communicate with each other. To provide a practical example, say you are using a laptop that is connected to a router. That laptop and other devices connected to the router are part of a web known as a computer network.

They can communicate with each other, but how?

As mentioned earlier a computer network allows computing devices to communicate with each other. In order for this to occur the computing devices on the network must speak the same language.

Similarly to how in order for humans to understand each other we must speak or understand the same language.

  • Computing devices on a computer network must use a common communications protocol (wiki.)
    • Communications protocol = set of rules that describe how information is to be exchanged between entities

The Internet

We've established that in a computer network you can communicate with other devices also in the same network. But, what if we want to communicate with devices outside our network?

The Internet is, to put it succinctly, a network of networks. This means its a system in which computer networks can communicate with each other. To be more specific, the Internet is a globally connected set of computer networks that all use a suite of common protocols.

Deeper look at the parts of a computer network

Hosts/Nodes are single computing devices attached to a network. Hosts can have different purposes in a network similarly to how humans have different purposes -- albeit hosts don't have nearly as much options. A host can act as a server, client, or both!

  • Server is a host that listens for inbound network connections and provides services to other hosts.
  • Client is a host that makes outbound network connections and requests services from network servers.

It's important to realize that any computer (host) can be a server as long as it has the appropriate software (instructions). Therefore, when thinking of a server don't think it's some magical computing device. As your own laptop can be a server if adjusted correctly.

Some computing devices are constructed specifically to act as servers as a normal laptop wouldn't be powerful or efficient enough to handle large requests. (wiki.)

A client makes a request to a server and a server replies with a response.

Communicating on and across networks

So far, we have looked at what a computer network is and the parts such as clients and servers. But we have yet to describe how these hosts communicate with each other on their own networks and across networks.

As mentioned earlier computing devices must speak the same language -- use a common communciations protocol. The Internet Protocol Suite is a standardized framework for communciation on the Internet. The IPS ensures that computing devices on a computer network speak the language.

So if a device wants to communicate on the internet it is most likely going to utilize the Internet Protocol Suite to do so. One should note that a name that is sometimes used to refer to the four layer network model the Internet Protocol Suite is designed around is called TCP/IP model.

Network Stacks

The TCP/IP model refers to the four layer model the Internet Protocl Suite is designed around but what is a network stack? A network stack is a layered model of network protocols.

A layer in a network stack reprsents a scope of responsibilities that are fulfilled by the protocols in the layer.

Protocols in the lowest layers interact with networking hardware. Protocols in intermediate layers provide services (routing, addressing). Protocols in the upper layers interact with applications.

Deeper look at the TCP/IP model

As mentioned earlier, the TCP/IP model is a four layer model that the IPS is designed around and network layers represent a scope of responsbilities that are fulfilled by the protocols in those layers. What are the responsbilities of the layers in the TCP/IP model?

The TCP/IP model is compromised of the following layers:

  • Application
    • Protocols provide application specific functionality by structuring the data used in process-to-process communication across a network.
    • Protocols used: HTTP, SSH
  • Transport
    • Protocols provide a communications channel for applications to send/receive data between hosts.
    • Protocols used: TCP, UDP
  • Internet
    • Protocols provide a way to communicate across networks. This layer is responsible for identifying hosts with addresses and enabling the routing of data from network to network across the internet.
    • Protocols used: IP
  • Link
    • Protocols provide a way to communicate on a local network.
    • Protocols used: Wi-Fi, Ethernet
